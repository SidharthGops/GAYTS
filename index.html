PostgreSQL Database Schema
Two-table design — vehicles (whitelist) + gate_logs (every detection)
vehicles Authorized plate whitelist
ColumnTypeKeyDescription
id SERIAL PK Auto-increment primary key
plate_number VARCHAR(20) Unique, indexed, uppercase
registered_at TIMESTAMP Auto-set on insert
gate_logs Automatic detection logbook
ColumnTypeKeyDescription
id SERIAL PK Auto-increment primary key
plate_number VARCHAR(20) OCR output — raw detected plate
status VARCHAR(15) AUTHORIZED or UNAUTHORIZED
confidence_score FLOAT OCR confidence 0.0–1.0
snapshot_path TEXT Saved frame file path
timestamp TIMESTAMP Auto-set on insert (IST)
🔁 AUTHORIZATION LOGIC
# For every OCR detection: vehicle = SELECT * FROM vehicles WHERE plate_number = 'KL07BH1234' if vehicle: status = "AUTHORIZED" # in whitelist ✓ else: status = "UNAUTHORIZED" # not in whitelist ✗ # Both cases → INSERT into gate_logs # Unauthorized: snapshot saved, alert raised
🔗 TABLE RELATIONSHIP
vehicles
id · plate_number · registered_at
↕ plate_number match (no FK — by design)
gate_logs
id · plate_number · status · confidence_score · snapshot_path · timestamp
Design decision:
No FK between tables — gate_logs stores the raw plate string from OCR. This means unauthorized detections (plates not in vehicles) are still fully logged without null foreign keys.
💡 USEFUL QUERIES
-- All logs SELECT * FROM gate_logs ORDER BY timestamp DESC; -- Only unauthorized attempts SELECT * FROM gate_logs WHERE status = 'UNAUTHORIZED' ORDER BY timestamp DESC; -- Count by status today SELECT status, COUNT(*) FROM gate_logs WHERE DATE(timestamp) = CURRENT_DATE GROUP BY status; -- Vehicles with most detections SELECT plate_number, COUNT(*) as hits FROM gate_logs GROUP BY plate_number ORDER BY hits DESC LIMIT 10;

AI / ML Pipeline
Camera → vehicle detection → plate localization → OCR → authorization — under 500ms
⚙️ PIPELINE STAGES
1
Video Frame Capture
RTSP/HTTP stream captured at 15–30 FPS via OpenCV. Frames resized to 640×640 and pushed to processing queue. Supports webcam for demo mode.
OpenCVRTSPasyncio
2
Vehicle Detection (YOLOv8)
YOLOv8n detects vehicles with bounding boxes. Only car/truck/motorcycle classes are processed. Frames with no vehicle are skipped immediately.
YOLOv8UltralyticsCUDA
3
License Plate Localization
Second YOLOv8 model fine-tuned on Indian plates finds the plate ROI within the vehicle bounding box. Perspective correction handles angled shots.
YOLOv8-LPwarpPerspectiveCustom dataset
4
Image Preprocessing
Plate ROI enhanced: grayscale → CLAHE contrast → morphological ops → Gaussian denoise. Handles IR/night camera input for low-light conditions.
OpenCV CLAHEPillow
5
OCR Extraction
EasyOCR (primary) + Tesseract (fallback) extract text. Results validated against Indian plate regex. Fuzzy matching corrects common OCR errors (0↔O, 1↔I, 8↔B).
EasyOCRTesseract 5regexfuzzywuzzy
6
Authorization + Log
Verified plate hits POST /api/authorize. Backend checks vehicles table, inserts into gate_logs, returns AUTHORIZED or UNAUTHORIZED. Gate command sent via MQTT.
FastAPIPostgreSQLMQTTESP32
